# 前端项目优化计划

## 🎯 优化目标

对前端项目进行全面的文件清理与配置统一管理，提升项目可维护性和安全性。

***

## 📋 任务清单

### 第一阶段：文件清理优化

#### 1.1 删除无用文件

* 删除 `src/components/business/__init__.py`

* 删除 `src/components/common/__init__.py`

* 删除 `src/components/layout/__init__.py`

* 删除 `src/data/mockData.js`（未被引用）

* 删除 `src/composables/useCommon.js`（未被引用）

#### 1.2 清理Git冲突标记

* 清理 `src/main.js` 中的Git冲突标记

* 清理 `src/router/index.js` 中的Git冲突标记

* 确保文件格式正确，功能正常

#### 1.3 文件引用关系分析

* 验证所有删除的文件确实未被引用

* 确保删除后项目功能不受影响

***

### 第二阶段：配置统一管理

#### 2.1 环境变量规范化

* **删除** **`.env`** **文件**（与development重复，且包含敏感信息）

* **更新** **`.env.development`**（开发环境配置）

  ```
  VITE_API_BASE_URL=http://127.0.0.1:3000/api
  VITE_WS_URL=ws://127.0.0.1:3000/ws
  VITE_APP_TITLE=WebScan AI Security Platform (Dev)
  VITE_APP_VERSION=1.0.0-dev
  VITE_DEBUG=true
  VITE_REQUEST_TIMEOUT=30000
  VITE_ENABLE_PERFORMANCE_MONITORING=false
  ```

* **创建** **`.env.test`**（测试环境配置）

  ```
  VITE_API_BASE_URL=http://test.example.com/api
  VITE_WS_URL=ws://test.example.com/ws
  VITE_APP_TITLE=WebScan AI Security Platform (Test)
  VITE_APP_VERSION=1.0.0-test
  VITE_DEBUG=true
  VITE_REQUEST_TIMEOUT=30000
  VITE_ENABLE_PERFORMANCE_MONITORING=false
  ```

* **更新** **`.env.production`**（生产环境配置）

  ```
  VITE_API_BASE_URL=/api
  VITE_WS_URL=/ws
  VITE_APP_TITLE=WebScan AI Security Platform
  VITE_APP_VERSION=1.0.0
  VITE_DEBUG=false
  VITE_REQUEST_TIMEOUT=30000
  VITE_ENABLE_PERFORMANCE_MONITORING=true
  ```

#### 2.2 迁移硬编码配置

* **更新** **`src/utils/api.js`**

  * 使用 `import.meta.env.VITE_API_BASE_URL`

  * 使用 `import.meta.env.VITE_REQUEST_TIMEOUT`

* **更新** **`src/views/Reports.vue`**

  * 使用 `import.meta.env.VITE_API_BASE_URL`

* **更新** **`src/components/business/POCScanForm.vue`**

  * 使用 `import.meta.env.VITE_REQUEST_TIMEOUT`

#### 2.3 更新 `.gitignore`

* 添加 `.env.local`（本地开发配置）

* 添加 `.env.test`（测试环境配置，如果不需要提交）

* 确保敏感信息不会被提交

***

### 第三阶段：验证与测试

#### 3.1 构建验证

* 运行 `npm run build` 确保构建成功

* 检查构建输出无错误

#### 3.2 功能验证

* 启动开发服务器 `npm run dev`

* 验证所有页面正常加载

* 验证API调用正常

* 验证环境变量正确加载

#### 3.3 代码质量检查

* 运行 `npm run lint` 检查代码规范

* 修复所有lint错误

***

### 第四阶段：生成优化报告

#### 4.1 清理文件列表

* 列出所有已删除的文件

* 说明删除原因

#### 4.2 配置迁移对照表

* 展示配置项迁移前后对比

* 说明每个配置项的用途

#### 4.3 项目体积对比

* 统计优化前的项目文件数量

* 统计优化后的项目文件数量

* 计算减少的文件数量和百分比

***

## ⚠️ 注意事项

1. 删除文件前会再次确认引用关系
2. 所有配置迁移会保持向后兼容
3. 敏感信息（如API密钥）将移至本地环境变量
4. 优化过程中会保留Git历史记录
