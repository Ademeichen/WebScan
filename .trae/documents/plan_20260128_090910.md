# 基于 Pocsuite3 和 Seebug 的智能 POC 验证系统实施计划

## 📋 项目概述

设计并实现一个独立的 POC 验证节点，集成到现有的 LangGraph 智能体工作流中，实现端到端的自动化 POC 验证流程。

## 🎯 核心目标

* 创建专用的 POC 验证节点，符合 LangGraph 节点规范

* 集成现有的 Pocsuite3Agent 和 Seebug API

* 实现多线程并发 POC 执行

* 提供完整的错误处理和重试机制

* 生成结构化的验证报告

## 🔧 详细实施步骤

### 阶段 1：数据模型与配置（1-2小时）

#### 1.1 扩展 AgentState

* 在 `state.py` 中添加 POC 验证相关字段：

  * `poc_verification_tasks`: 待验证的 POC 任务列表

  * `poc_verification_results`: POC 验证结果列表

  * `poc_execution_stats`: POC 执行统计信息

  * `poc_verification_status`: 验证状态（pending/running/completed/failed）

#### 1.2 创建 POC 验证数据模型

* 创建 `models/poc_verification.py`，包含：

  * `POCVerificationTask`: POC 验证任务记录

  * `POCVerificationResult`: POC 验证结果记录

  * `POCExecutionLog`: POC 执行日志记录

#### 1.3 扩展配置文件

* 在 `config.py` 中添加：

  * `POC_VERIFICATION_ENABLED`: 是否启用 POC 验证

  * `POC_MAX_CONCURRENT_EXECUTIONS`: 最大并发执行数

  * `POC_EXECUTION_TIMEOUT`: POC 执行超时时间

  * `POC_RETRY_MAX_COUNT`: 最大重试次数

  * `POC_RESULT_ACCURACY_THRESHOLD`: 结果准确率阈值

### 阶段 2：POC 系统核心模块（4-6小时）

#### 2.1 POC 脚本管理器（poc\_manager.py）

* 功能：

  * 从 Seebug 同步 POC 脚本

  * 本地自定义 POC 脚本加载

  * POC 版本控制

  * POC 缓存管理

  * POC 元数据管理（名称、类型、严重度等）

#### 2.2 目标信息管理器（target\_manager.py）

* 功能：

  * 批量目标导入（CSV、JSON、Excel）

  * 单个目标输入

  * 从智能体工作流接收目标数据

  * 目标去重和验证

  * 目标分组管理

#### 2.3 POC 验证执行引擎（verification\_engine.py）

* 功能：

  * 多线程并发 POC 执行

  * 资源限制管理（CPU、内存、网络）

  * 执行超时控制

  * 错误重试机制

  * 执行进度监控

  * 实时日志捕获

#### 2.4 结果分析器（result\_analyzer.py）

* 功能：

  * 漏洞验证状态判定

  * 漏洞等级评估（CVSS 评分）

  * 误报检测

  * 结果置信度计算

  * 异常结果标记

#### 2.5 报告生成器（report\_generator.py）

* 功能：

  * HTML 格式报告（带图表）

  * JSON 格式报告（结构化数据）

  * PDF 格式报告（可打印）

  * 执行摘要生成

  * 漏洞详情列表

  * 统计信息（成功率、耗时等）

### 阶段 3：LangGraph 节点集成（2-3小时）

#### 3.1 创建 POC 验证节点

* 创建 `ai_agents/core/poc_verification_node.py`

* 实现 `POCVerificationNode` 类：

  * 符合 LangGraph 节点规范（`__call__` 方法）

  * 接收 `AgentState` 作为输入

  * 返回更新后的 `AgentState`

  * 节点状态管理（暂停、继续、终止）

  * 与其他节点的数据传递

#### 3.2 集成到 LangGraph 工作流

* 修改 `ai_agents/core/graph.py`：

  * 导入 `POCVerificationNode`

  * 在工作流中添加 POC 验证节点

  * 定义条件边（根据验证结果决定下一步）

  * 配置节点执行顺序

### 阶段 4：API 路由与接口（2-3小时）

#### 4.1 创建 POC 验证 API

* 创建 `api/poc_verification.py`：

  * `POST /api/poc/verification/tasks`: 创建验证任务

  * `GET /api/poc/verification/tasks`: 列出验证任务

  * `GET /api/poc/verification/tasks/{task_id}`: 获取任务详情

  * `POST /api/poc/verification/tasks/{task_id}/pause`: 暂停任务

  * `POST /api/poc/verification/tasks/{task_id}/resume`: 继续任务

  * `POST /api/poc/verification/tasks/{task_id}/cancel`: 取消任务

  * `GET /api/poc/verification/tasks/{task_id}/results`: 获取验证结果

  * `GET /api/poc/verification/tasks/{task_id}/report`: 生成报告（支持 html/json/pdf）

#### 4.2 注册 API 路由

* 修改 `api/__init__.py`：

  * 导入 `poc_verification` 路由

  * 添加到主路由器
