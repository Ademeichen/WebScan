# 后端API问题分析与前端修复计划

## 问题诊断

### 控制台错误分析

1. **ReferenceError: router is not defined** - 某个组件中的事件处理器使用了router但未导入
2. **ReferenceError: formatDate is not defined** - 多个组件使用了formatDate但未正确导入
3. **TypeError: Cannot read properties of undefined (reading 'severity')** - VulnerabilityCard组件在vulnerability为undefined时尝试访问severity属性
4. **Request failed with status code 500** - 后端API返回500错误
5. **加载POC类型失败** - POC类型API加载失败

### 根本原因

#### 前端问题

1. **VulnerabilityResults.vue** (第329行)
   - 在return中导出了`formatDate`，但没有在setup函数中导入
   - 缺少 `import { formatDate } from '@/utils/date'`

2. **VulnerabilityCard.vue** (第28行, 第134行)
   - 使用了`formatDate`但没有导入
   - 返回了未定义的`Document`变量
   - 在vulnerability为undefined时访问severity属性导致错误

3. **Profile.vue** (第48行, 第52行)
   - 使用了`formatDate`但没有导入

#### 后端问题

1. **POC API** - `/poc/types` 端点实现正确，但可能存在导入问题
2. **Tasks API** - `/tasks/{task_id}/vulnerabilities` 可能因数据库查询失败返回500

## 修复计划

### 第一步：修复前端组件导入问题

1. **VulnerabilityResults.vue**
   - 添加 `import { formatDate } from '@/utils/date'`

2. **VulnerabilityCard.vue**
   - 添加 `import { formatDate } from '@/utils/date'`
   - 移除return中的`Document`变量
   - 添加vulnerability的空值检查

3. **Profile.vue**
   - 添加 `import { formatDate } from '@/utils/date'`

### 第二步：增强VulnerabilityCard组件健壮性

- 添加vulnerability对象的空值检查
- 在访问vulnerability属性前验证对象是否存在
- 提供默认值以避免undefined错误

### 第三步：检查并修复后端API

1. **检查POC API导入问题**
   - 验证POC_FUNCTIONS字典的导入
   - 确保所有POC函数正确导入

2. **检查Tasks API错误处理**
   - 验证数据库查询的异常处理
   - 添加更详细的日志记录

3. **检查Notifications API**
   - 验证MOCK_NOTIFICATIONS数据结构
   - 确保所有通知对象包含必需字段

### 第四步：测试验证

1. 测试通知列表和详情页面
2. 测试用户资料页面
3. 测试漏洞结果页面
4. 测试POC类型加载
5. 验证所有API端点正常工作

## 预期结果

- 所有前端组件正确导入所需函数
- VulnerabilityCard组件能正确处理空值情况
- 后端API返回正确的响应格式
- 控制台错误全部消除
- 所有页面功能正常工作
