# nodes.py与new_nodes.py合并及新增节点纳入图实施方案

## 📋 任务概述

将`nodes.py`（原有5个节点）与`new_nodes.py`（新增5个节点）系统性合并，实现新增节点正确纳入LangGraph工作流图中，确保完整的自主扫描能力。

## 🔍 问题分析

### 当前问题
1. **节点未纳入图**：`graph.py`中`_build_graph()`方法只添加了5个原有节点，新增的5个节点（环境感知、代码生成、代码执行、功能补充、智能决策）未被添加
2. **入口点错误**：工作流入口仍为"task_planning"，应该是"environment_awareness"
3. **缺少条件分支**：没有实现智能决策分支（固定工具 vs 代码生成 vs 功能增强）
4. **可视化图不匹配**：`visualize()`方法的Mermaid图包含了新增节点，但实际图结构不包含

### 需要合并的节点
**原有节点（5个）**：
- TaskPlanningNode - 任务规划
- ToolExecutionNode - 工具执行
- ResultVerificationNode - 结果验证
- VulnerabilityAnalysisNode - 漏洞分析
- ReportGenerationNode - 报告生成

**新增节点（5个）**：
- EnvironmentAwarenessNode - 环境感知
- CodeGenerationNode - 代码生成
- CapabilityEnhancementNode - 功能补充
- CodeExecutionNode - 代码执行
- IntelligentDecisionNode - 智能决策

## 🎯 实施计划

### 阶段一：代码合并（保持向后兼容）

#### 1.1 合并nodes.py和new_nodes.py
- **操作**：将`new_nodes.py`中的5个节点类合并到`nodes.py`
- **策略**：
  - 保持原有节点类不变
  - 在`nodes.py`末尾添加新增节点类
  - 更新`__all__`导出列表（如果存在）
  - 添加详细的中文注释说明节点用途
- **输出**：合并后的`nodes.py`包含10个节点类

#### 1.2 更新graph.py导入
- **操作**：修改`graph.py`的导入语句
- **变更**：
  ```python
  # 从
  from .nodes import (原有5个节点)
  from .new_nodes import (新增5个节点)
  
  # 改为
  from .nodes import (所有10个节点)
  ```

### 阶段二：图结构重构（实现完整工作流）

#### 2.1 重构_build_graph()方法
- **操作**：完全重写`_build_graph()`方法
- **添加所有节点**（10个）：
  ```python
  workflow.add_node("environment_awareness", self.env_awareness_node)
  workflow.add_node("task_planning", self.planning_node)
  workflow.add_node("intelligent_decision", self.intelligent_decision_node)
  workflow.add_node("tool_execution", self.execution_node)
  workflow.add_node("code_generation", self.code_generation_node)
  workflow.add_node("code_execution", self.code_execution_node)
  workflow.add_node("capability_enhancement", self.capability_enhancement_node)
  workflow.add_node("result_verification", self.verification_node)
  workflow.add_node("vulnerability_analysis", self.analysis_node)
  workflow.add_node("report_generation", self.report_node)
  ```

#### 2.2 调整入口点
- **操作**：将入口点从"task_planning"改为"environment_awareness"
  ```python
  workflow.set_entry_point("environment_awareness")
  ```

#### 2.3 实现条件分支
- **添加分支判断函数**：
  - `_decide_scan_type()` - 智能决策：选择扫描类型
  - `_code_execution_result()` - 判断代码执行结果
  - `_should_continue()` - 判断是否继续执行工具

- **添加条件边**：
  ```python
  # 智能决策分支
  workflow.add_conditional_edges(
      "intelligent_decision",
      self._decide_scan_type,
      {
          "fixed_tool": "tool_execution",
          "custom_code": "code_generation",
          "enhance_first": "capability_enhancement"
      }
  )
  
  # 代码执行结果分支
  workflow.add_conditional_edges(
      "code_execution",
      self._code_execution_result,
      {
          "success": "result_verification",
          "need_enhance": "capability_enhancement"
      }
  )
  ```

#### 2.4 定义流程边
- **基础流程**：环境感知 → 任务规划 → 智能决策
- **固定工具流程**：智能决策 → 工具执行 → 结果验证 → 循环/分析
- **代码生成流程**：智能决策 → 代码生成 → 代码执行 → 结果验证/功能补充
- **功能补充闭环**：功能补充 → 代码执行（重试）
- **后续流程**：漏洞分析 → 报告生成 → 结束
