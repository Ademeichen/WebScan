# 前后端API全面检查与修复计划

## 🔴 问题1：Seebug_Agent导入失败

**根本原因**：`Seebug_Agent/__init__.py` 没有导出 `SeebugAgent` 类

**修复方案**：
- 修改 `Seebug_Agent/__init__.py`，添加 `SeebugAgent` 到 `__all__` 列表
- 同时需要添加 `from .main import SeebugAgent`

## 🔴 问题2：前端环境变量配置冲突

**根本原因**：`.env.development` 文件包含Git合并冲突标记

**修复方案**：
- 清理 `.env.development` 文件中的Git冲突标记（<<<<<<< HEAD, =======, >>>>>>>）
- 统一API基础URL为 `http://127.0.0.1:3000/api`（与后端默认端口3000一致）
- 更新 `.env` 文件的API基础URL为 `http://127.0.0.1:3000/api`

## 🔴 问题3：前端API路径与后端不匹配

**发现的不匹配**：
- 前端 `awvsApi.startScan` 调用 `/awvs/scans`，但后端路由是 `/awvs/scan`
- 前端 `awvsApi.getScans` 调用 `/awvs/scans`，后端是 `/awvs/scans` ✅
- 前端 `awvsApi.createTarget` 调用 `/awvs/targets`，后端是 `/awvs/target` ❌

**修复方案**：
- 修改 `front/src/utils/api.js` 中的API路径，使其与后端路由完全匹配
- 更新 `awvsApi.startScan` 的URL为 `/awvs/scan`
- 更新 `awvsApi.createTarget` 的URL为 `/awvs/target`

## 🔴 问题4：Vite代理配置优化

**修复方案**：
- 移除 `vite.config.js` 中的 `host: true`（避免启动时显示多个IP地址）
- 简化配置，仅保留必要的API代理设置

## 🔴 问题5：AWVSScanForm组件API调用问题

**修复方案**：
- 修改 `AWVSScanForm.vue` 中的API调用，使用正确的数据格式
- 确保提交的数据结构与后端 `/awvs/scan` 接口匹配
